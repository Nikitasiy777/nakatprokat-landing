# CLAUDE.md — Контекст проекта: Накат Прокат

> Читай этот файл первым в каждом новом чате.

## Что за проект

Лендинг для **Накат Прокат** — официального партнёра Яндекс Маркета и Самоката в Казани.  
Задача: привлекать курьеров, убеждать оставить заявку.  
Главное УТП: **велосипед бесплатно на первую неделю** — больше ни у кого нет.

## Навигация по файлам

| Файл | Содержание |
|------|-----------|
| `.design` | **Дизайн-система: цвета, шрифты, компоненты, стек — читать перед любым фронтендом** |
| `.business/INDEX.md` | Главный индекс всей бизнес-документации |
| `.business/company.md` | О компании: история, миссия, команда |
| `.business/audience.md` | Целевая аудитория, портреты, боли |
| `.business/products.md` | Продукт: что предлагаем курьерам |
| `.business/goals.md` | Цели проекта, KPI лендинга |
| `.business/economics.md` | Экономика: CPL, бюджет, воронка |
| `.business/competitors.md` | Анализ конкурентов, их УТП, структура |
| `Plans/` | Планы разработки, фазы, статусы |

## Стек

- Next.js 15 + App Router
- Tailwind CSS + TypeScript
- Framer Motion, Lucide React
- Bebas Neue (заголовки) + Inter (текст)
- Хостинг: Vercel

## Правила для агента

1. Перед любым решением — прочитай `.business/INDEX.md`
2. Перед любой работой над фронтендом — прочитай `.design`
3. При изменении бизнес-логики — обновляй `.business/` файлы
4. Новые планы сохраняй в `Plans/`
5. Все решения по дизайну опирай на `audience.md` (боли ЦА)
6. УТП берётся из `products.md` и `competitors.md`
7. Название компании — **Накат Прокат**. Нигде не использовать "Эдем".
6. **Все ответы писать только на русском языке**
7. **После каждой выполненной задачи обязательно писать итог в формате:**

---
### Итог задачи
**Что просили:** ...  
**Что выполнено:** ...  
**Что не выполнено / осталось:** ...  
---

8. **После каждой выполненной задачи давать звуковое оповещение** командой:
   `afplay /System/Library/Sounds/Glass.aiff`

## Управление контекстом (обязательно)

9. **Никогда не читать `node_modules/`, `.next/`, `dist/`** — они в `.claudeignore`.
10. **Читать только нужные файлы** — не делать `find` или `ls` по всему проекту без необходимости.
11. **При длинном чате (более 15–20 обменов сообщениями)** — предложить пользователю выполнить `/compact`, написав:
    > "Чат стал длинным — рекомендую сжать контекст командой `/compact`, чтобы не было ошибок."
12. **Если получаешь ошибку о превышении контекста** — сразу написать пользователю: "Пожалуйста, нажми `/compact` в строке ввода и потом повтори запрос."
